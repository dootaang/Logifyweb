# 로그제조기 올인원 (Next.js)

모든 설정을 한 곳에서 관리하는 스마트 웹로그 생성기입니다.

## ✨ 주요 기능

- 🎨 **5가지 로그제조기 타입** - 북마클릿형, 배너형, 제리형, 챗챈형, 빙둔형
- 🖼️ **스마트 이미지 처리** - 프록시 서버를 통한 이미지 최적화
- 📱 **반응형 디자인** - 모든 디바이스에서 완벽한 사용성
- 🌙 **다크/라이트 테마** - 시스템 테마 자동 감지
- 💾 **설정 자동 저장** - localStorage를 통한 설정 영구 보존
- 📋 **원클릭 복사** - 생성된 HTML 코드 클립보드 복사
- 🔄 **단어 교환** - 자동 텍스트 치환 기능
- 🎯 **실시간 미리보기** - 설정 변경 즉시 반영

## 🛠️ 사용 방법

### 1. 로그제조기 타입 선택
- 상단의 5가지 타입 중 원하는 스타일 선택
- 각 타입별로 고유한 디자인과 특성

### 2. 본문 내용 작성
- 텍스트 에리어에 로그 내용 입력
- 대화는 큰따옴표(""), 생각은 작은따옴표('') 사용
- 문단 구분은 빈 줄로 표시

### 3. 이미지 설정
- **배경 이미지 URL 입력** 또는 **기본 이미지 선택**
- 권장 사이즈: 1400px × 400px (3.5:1 비율)
- 이미지 HTML 태그 붙여넣기 지원 (자동 URL 추출)

### 4. 텍스트 및 색상 설정
- 왼쪽/오른쪽 텍스트 설정
- 본문 배경색/글자색 조정
- 큰따옴표/작은따옴표 색상 설정
- 그라데이션 효과 활성화

### 5. 스타일 옵션
- ✅ 큰따옴표 색상 활성화
- ✅ 큰따옴표 그라데이션 효과
- ✅ 큰따옴표 볼드체
- ✅ 작은따옴표 기울기
- ✅ 문단 들여쓰기

### 6. 텍스트 조절
- 폰트 크기: 10px ~ 24px
- 줄 간격: 1.2 ~ 2.5

### 7. 단어 교환
- 최대 3개의 단어 치환 규칙 설정
- 정규표현식 지원

### 8. HTML 복사
- "📋 HTML 복사" 버튼으로 결과물 복사
- 아카라이브, 디시인사이드 등에 바로 붙여넣기

## 🖼️ 이미지 업로드 설정

### 아카라이브 이미지 업로드

프로젝트는 아카라이브 이미지 업로드를 지원합니다. 두 가지 방법이 있습니다:

#### 방법 1: 직접 업로드 (권장)
1. [아카라이브 게시글 작성](https://arca.live/b/characterai/write)으로 이동
2. 이미지를 드래그&드롭 또는 클릭하여 업로드
3. 에디터에 삽입된 이미지의 HTML 코드를 복사
4. 로그제조기의 "아카라이브 이미지 URL" 필드에 붙여넣기
5. URL이 자동으로 추출되어 적용됩니다

#### 방법 2: 프록시 서버 설정 (고급)
자동 이미지 업로드를 위해 프록시 서버를 설정할 수 있습니다:

1. 프로젝트 루트에 `.env.local` 파일 생성:
```bash
# 아카라이브 이미지 업로드를 위한 프록시 서버 URL
NEXT_PUBLIC_PROXY_URL=https://your-proxy-server.com
```

2. 프록시 서버는 다음 인터페이스를 구현해야 합니다:
```typescript
// POST /upload
interface UploadRequest {
  upload: File; // FormData 필드명
}

interface UploadResponse {
  url: string; // 아카라이브 형식 URL (//ac-p1.namu.la/...)
}
```

3. 프록시 서버 예시 기능:
   - 클라이언트로부터 파일 수신
   - 아카라이브 API로 파일 전송
   - 아카라이브 URL 반환

### 지원 이미지 형식
- **파일 형식**: JPG, PNG, GIF
- **최대 크기**: 5MB
- **권장 사이즈**: 1400px × 400px (3.5:1 비율)

## 🔍 트러블슈팅

### Q: 이미지가 로드되지 않아요
A: 이미지 프록시 서버(images.weserv.nl)를 사용하여 CORS 문제를 해결합니다. 네트워크 연결을 확인해주세요.

### Q: 이미지 업로드 시 "아카라이브 업로드 실패: 404" 오류가 발생해요
A: 프록시 서버가 아카라이브 API에 접근할 때 404 오류가 발생하는 경우입니다.

**원인:**
- 아카라이브 API 엔드포인트 변경
- 프록시 서버의 API 호출 방식 문제
- 아카라이브 서버 일시적 문제

**해결방법:**
1. **직접 업로드 사용 (권장)**:
   - [아카라이브 글쓰기](https://arca.live/b/characterai/write)에서 이미지 업로드
   - 생성된 HTML 태그를 복사하여 "아카라이브 이미지 URL" 필드에 붙여넣기
   
2. **잠시 후 재시도**:
   - 아카라이브 서버 문제일 경우 몇 분 후 다시 시도
   
3. **기본 이미지 사용**:
   - 유즈, 레몬, 현대밤 등 내장된 기본 이미지 활용

### Q: 설정이 저장되지 않아요
A: 브라우저의 localStorage가 활성화되어 있는지 확인하세요. 시크릿 모드에서는 설정이 저장되지 않습니다.

### Q: 다크모드가 적용되지 않아요
A: 브라우저의 시스템 테마 설정을 확인하세요. CSS 변수를 지원하는 최신 브라우저를 사용해주세요.

### Q: HTML 복사가 안 돼요
A: 브라우저의 클립보드 권한을 확인하세요. HTTPS 환경에서만 정상 작동합니다.
